# TrueMatch Relational Database Design

## Overview

TrueMatch is a relational database designed for a hypothetical dating platform. This project demonstrates the design and implementation of a normalized MySQL database capable of supporting core application functionality while maintaining data integrity and scalability.

The database was designed to manage user accounts, profile information, matching preferences, messaging, premium subscriptions, safety reporting, and event participation through a structured relational model.

---

## Business Objective

Design a scalable relational database that efficiently stores and manages user data while supporting common business operations for a dating platform. The project emphasizes sound database design principles, normalization, and SQL implementation to ensure accuracy, consistency, and efficient querying.

---

## Features

- Normalized relational database design
- Conceptual, logical, and physical ER diagrams
- Primary and foreign key relationships
- Many-to-many relationship implementation
- SQL database implementation using MySQL
- Business-focused data modeling
- Sample analytical SQL queries
